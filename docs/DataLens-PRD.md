# DataLens MVP PRD - 产品需求文档

> 版本：1.0-final | 日期：2026-04-02 | 作者：产品组（老莫/老牛/老衲）
> 技术选型已确认：Tauri v2 + DuckDB | 垂直场景已确认：电商运营

---

## 1. 产品概述

### 1.1 产品定义

**产品名称**：DataLens（数据透镜）

**一句话定义**：桌面端本地数据分析工作台——拖入数据文件，用自然语言提问，AI 自动生成分析和可视化，所有数据不离开本机。

**一句话差异化定位**：唯一一个数据不离开你电脑的 AI 数据分析工具。

**核心价值主张**：让不会写 SQL 的人也能在 5 分钟内从数据中获得洞察，且无需担心数据泄露。

### 1.2 为什么是桌面端

| 桌面端优势 | 对 DataLens 的意义 |
|------------|-------------------|
| 本地文件直接访问 | 拖拽导入 CSV/Excel，无需上传 |
| 本地计算性能 | 处理百万行数据不受网络带宽限制 |
| 本地 LLM 支持 | 连接 Ollama/LM Studio，数据零泄漏 |
| 离线可用 | 无网络环境也能完成数据分析 |
| 系统级集成 | 连接本地数据库（SQLite/PostgreSQL） |

### 1.3 MVP 范围

**做**：CSV 导入 + 自然语言查询 + 图表可视化 + 电商场景模板
**不做**：Excel/JSON 导入、数据库连接、多数据源联合分析、团队协作、报告导出

### 1.4 核心竞争壁垒

1. **隐私即功能**：不是"也能本地"，而是"只在本地"。这是产品基因，不是附加功能。大厂的商业模式决定了他们不会主推本地方案
2. **垂直场景模板**：针对电商运营预置分析模板（渠道 ROI、SKU 分析、复购率漏斗、广告 ACOS），开箱即用。通用工具做不到这种垂直深度
3. **本地大数据处理**：基于 DuckDB 嵌入式引擎，本地处理百万行 CSV 秒级响应，Web 应用做不到的体验

## 2. 目标用户画像

### 2.1 主要用户：电商运营（MVP 垂直场景）

**用户 A — 小李，电商运营主管**
- 28 岁，某 DTC 品牌运营主管，团队 3 人
- 日常：每天从 Shopify/抖店后台导出 CSV，分析各渠道 ROI、SKU 表现、退货率
- 现状痛点：Excel 透视表勉强能用，但一遇到"按周对比各渠道新客占比变化趋势"这种多维分析就抓瞎。每次大促后要花整整一天做复盘报告
- 付费意愿：$15/月毫无压力，比招个数据实习生便宜 100 倍
- 关键需求：拖进 CSV → 说人话提问 → 出图表 → 导出报告发老板

**用户 B — 王姐，独立电商卖家**
- 35 岁，亚马逊 + 独立站卖家，一人运营
- 日常：手动在 Excel 里算利润率、库存周转、广告 ACOS
- 现状痛点：不会公式，每个月花半天手动算，经常算错。想知道"哪个 SKU 该补货"要翻好几个表格交叉对比
- 付费意愿：能帮她少犯一次补货决策错误就值回一年费用
- 关键需求：简单到极致，拖进去就能问，不要让我配置任何东西

### 2.2 次要用户（V1.0 扩展）

**用户 C — 张总，SaaS 公司产品经理**
- 数据团队排期紧，简单分析不想等。常见场景：导出用户行为日志 CSV，想快速看某功能的留存漏斗
- 对"数据不上云"很敏感（公司合规要求）

**用户 D — 刘老师，高校研究员**
- 实验数据分析、论文图表生成。需要导出高质量图表
- 对价格敏感，但学术版/教育折扣可打动

## 3. MVP 用户故事

### US-1：导入数据文件

> 作为电商运营，我想把从淘宝后台导出的 CSV 文件拖进 DataLens，自动识别字段含义，这样我不用手动配置就能开始分析。

**验收标准**：
- 支持拖拽或点击选择 CSV 文件
- 自动检测编码（UTF-8/GBK/GB2312/Latin1）并正确解码，乱码时弹出预览让用户确认
- 自动识别字段类型（数值/日期/文本/分类）
- 导入后展示数据预览（前 20 行 + 字段统计摘要）
- 支持文件大小 <= 200MB，行数 <= 100 万行

### US-2：自然语言提问

> 作为电商运营，我想用中文问"上个月各渠道的付费转化率是多少"，DataLens 直接给我答案和图表，这样我不需要学 SQL。

**验收标准**：
- 输入框支持中文自然语言输入
- AI 将自然语言转换为 DuckDB SQL 查询
- 显示 AI 生成的 SQL（可折叠），用户可验证逻辑
- 查询执行时间 <= 5 秒（百万行以内）
- 支持追问和修正（"换成按周看"、"去掉退货订单"）
- SQL 执行失败时自动重试一次，二次失败给友好提示

### US-3：查看可视化结果

> 作为电商运营，我想看到直观的图表而不是一堆数字，这样我能快速理解数据趋势。

**验收标准**：
- AI 根据查询结果自动选择合适的图表类型（折线图/柱状图/饼图/表格）
- 图表支持鼠标悬停查看具体数值
- 用户可手动切换图表类型（图表右上角常驻切换按钮）
- 结果区域同时展示图表和数据表格（可切换）
- 数值自动格式化：金额保留 2 位小数，百分比保留 1 位

### US-4：使用预置问题快速分析

> 作为电商运营，导入数据后我不知道该问什么，希望 DataLens 能推荐最常用的分析问题，让我一键点击就能看结果。

**验收标准**：
- 导入 CSV 后，根据字段特征自动推荐 5-10 个预置问题
- 电商场景有专属问题模板（GMV 趋势、渠道转化、退货分析等）
- 问题以卡片形式展示，分类排列，每张卡片预览产出类型（如"→ 生成销售趋势折线图"）
- 点击预置问题直接执行分析，无需手动输入

### US-5：首次体验引导

> 作为新用户，我第一次打开 DataLens 时希望 30 秒内就能看到它的价值，这样我才有动力继续用。

**验收标准**：
- 内置电商示例 CSV 数据集（约 5000 行模拟数据）
- 首次打开自动进入引导流程，提供"先试试看"按钮加载示例数据
- 加载示例数据后自动触发第一个分析（"最近 30 天销售趋势"），用户无需任何操作就能看到图表
- 完整引导流程 <= 3 分钟

### US-6：配置 LLM 后端

> 作为用户，我想选择使用本地 LLM（Ollama）或云端 API（OpenAI），这样我可以在隐私和质量之间做取舍。

**验收标准**：
- 设置页面支持配置 OpenAI-compatible API 地址和 Key
- 自动检测连接是否成功
- 预设 Ollama 本地默认配置（localhost:11434）
- 明确展示本地 vs 云端的准确率差异
- 提供"网络活动日志"，用户可查看所有外发请求

## 6. 电商场景预置问题设计

导入电商数据后，DataLens 根据数据字段自动推荐高频分析问题，以卡片形式展示，用户点击即执行。

### 6.1 销售数据分析（适用数据源：订单导出 CSV）

**基础概览类：**
- 最近 30 天总销售额/订单量/客单价趋势 → 折线图
- 本月 vs 上月销售额对比，增长率多少 → 柱状对比图
- 销售额 TOP10 商品是哪些 → 横向柱状图

**商品分析类：**
- 哪些商品销量高但利润率低（需要调价） → 散点图
- 哪些商品退货率最高，退货原因分布 → 柱状图 + 饼图
- SKU 动销率分布——有多少商品 30 天内零销售 → 直方图

**时间维度类：**
- 一周中哪天销售最好，一天中哪个时段下单最多 → 热力图/柱状图
- 是否有明显的销售周期性（周末/月初/节假日） → 折线图

**客户分析类：**
- 新客 vs 老客订单占比和客单价差异 → 分组柱状图
- 复购率是多少，复购周期中位数 → 数值卡片 + 直方图
- 高价值客户（累计消费 TOP10%）的购买特征 → 数据表格

### 6.2 流量数据分析（适用数据源：平台流量报表/Google Analytics 导出）

- 各流量渠道的转化率对比 → 柱状图
- 访客到下单的流失漏斗（浏览→加购→下单→付款） → 漏斗图
- 跳出率最高的商品页面是哪些 → TOP N 表格

### 6.3 库存数据分析（适用数据源：库存管理导出）

- 当前库存按销售速度还能卖多少天 → 数据表格
- 哪些商品即将断货（库存 < 7 天销量） → 预警表格
- 哪些商品积压严重（库存 > 90 天销量） → 预警表格

### 6.4 预置问题的智能匹配

- 导入数据后，系统自动识别列名（如"订单金额""SKU""下单时间"），匹配最相关的问题模板
- 未识别的数据仍展示通用问题（TOP N、分布、趋势）
- 用户也可以直接输入自定义问题，预置问题只是快速入口

## 7. 首次体验流程设计（FTUE）

### 7.1 设计原则

- **30 秒内看到价值**——用户第一次打开到看到第一个分析结果不超过 30 秒
- **零配置启动**——不要上来就让填 API Key、选模型、配数据库
- **示例数据兜底**——如果用户没有自己的数据，用内置示例数据演示能力

### 7.2 完整流程

**Step 1：欢迎页（5 秒）**
- 一句话说清产品："导入数据，用中文提问，秒出图表"
- 两个按钮：
  - 【导入我的数据】→ 进入 Step 2
  - 【先试试看（示例数据）】→ 跳到 Step 3，加载内置电商示例数据集

**Step 2：数据导入（10 秒）**
- 拖拽 CSV 文件到窗口中央的大号放置区
- 导入后自动展示：文件名、行数、列名预览、数据类型识别结果
- 如果识别出是电商数据，显示绿色标签"已识别为电商销售数据"
- 底部按钮：【开始分析】

**Step 3：推荐问题 + 首次分析（15 秒）**
- 左侧：根据数据类型推荐的预置问题卡片
- 右侧：空白画布区域
- **自动触发第一个分析**——不等用户点击，直接运行"最近 30 天销售趋势"并在右侧渲染折线图
- 图表下方浮现提示："这就是 DataLens——你也可以用自己的话提问任何问题"

**Step 4：引导自定义提问**
- 底部输入框高亮闪烁，placeholder 文字："试试问：哪个商品退货率最高？"
- 用户输入第一个自定义问题后，响应结果，完成首次体验闭环

### 7.3 示例数据集设计

内置一份模拟电商数据（约 5000 行），包含字段：
- 订单 ID、下单时间、商品名、SKU、类目、数量、单价、订单金额
- 支付状态、退货状态、客户 ID、客户类型（新/老）、流量渠道

数据特征要求：
- 时间跨度 3 个月
- 包含周末波峰和一次促销活动数据
- 某个类目有明显增长趋势
- 某个商品退货率异常高
- 有一个明显的促销峰值

这样用户即使没带自己的数据，也能体验完整分析能力，产生"我要用自己的数据试试"的冲动。

## 8. 技术约束与限制

### 8.1 MVP 技术选型（已确认）

| 层次 | 技术 | 理由 |
|------|------|------|
| 桌面框架 | Tauri v2 | 包体积小（~10MB），内存低，Rust 后端性能强 |
| 前端 | React 19 + Vite + Tailwind | 复用现有技术栈 |
| 状态管理 | Zustand + Immer | 复用现有封装 |
| 数据引擎 | DuckDB (Rust binding) | 列式存储，OLAP 性能极强，原生 CSV 读取 |
| 可视化 | ECharts 或 Observable Plot | 图表类型丰富，中文支持好 |
| LLM 集成 | OpenAI SDK | 复用现有封装，兼容本地/云端 |

### 8.2 数据处理约束

| 约束 | MVP 限制 | 说明 |
|------|----------|------|
| 文件格式 | 仅 CSV | V1.0 扩展 Excel/JSON |
| 文件大小 | <= 200MB | 超出提示缩减或导入前 N 行 |
| 行数上限 | <= 100 万行 | 图表渲染需降采样 |
| 列数上限 | <= 100 列 | 超出后仅加载前 100 列并提示 |
| 编码支持 | UTF-8, GBK, Latin1 | 自动检测 + 用户确认 |

### 8.3 AI 查询能力边界

**能做**：
- 聚合统计：求和、计数、平均、最大最小、分组统计
- 筛选过滤：按时间范围、按字段值、按条件组合
- 排序和 Top-N 查询
- 时间序列分析（按日/周/月趋势）
- 对比分析（A vs B，如"新客 vs 老客客单价"）
- 占比分析（各类目销售占比）
- 基础计算字段（转化率 = 付款数/访客数）

**不能做（MVP 明确不支持）**：
- 预测/预估（"下个月能卖多少"）→ 明确告知不支持
- 因果推断（"为什么退货率上升"）→ 可给数据分布，不做归因
- 跨文件关联分析（暂不支持多文件 JOIN）
- 复杂统计模型（回归、聚类、异常检测）
- 实时数据（不支持 API/数据库实时连接）
- 复杂窗口函数

### 8.4 本地 LLM vs 云端 LLM

| 维度 | 本地模式 | 云端模式 |
|------|---------|---------|
| 隐私 | 数据 100% 不出本机 | 仅发送 Schema + 问题，不传原始数据 |
| 推荐模型 | Qwen2.5-7B-Instruct（需 8GB+ 显存或 16GB+ 内存） | GPT-4o / Claude Sonnet |
| 查询准确率 | ~75%（复杂查询可能理解偏差） | ~92% |
| 响应速度 | 3-10 秒（取决于硬件） | 1-3 秒 |
| 成本 | 免费 | 约 0.02-0.1 元/次查询 |
| 首次配置 | 需下载模型（约 4-8GB），引导式安装 | 填入 API Key 即可 |

**默认策略**：首次启动引导用户选择模式。推荐有条件的用户用本地模式，否则先用云端体验再考虑切本地。

### 8.5 数据隐私承诺

**明确承诺**：
- 用户数据文件永远存储在本地，应用不会上传原始数据到任何服务器
- 本地模式下，所有 AI 推理在本机完成，零网络传输
- 云端模式下，仅将表结构（列名 + 类型）和用户问题发送给 LLM API，**不传输任何行级数据**
- 应用不含任何数据采集/埋点 SDK（MVP 阶段），不收集用户行为数据

**技术实现**：
- 数据导入后存入本地 DuckDB，查询在本地执行
- 云端模式 LLM 调用流程：用户问题 + 表 Schema → LLM 生成 SQL → 本地执行 SQL → 本地渲染结果
- LLM 永远看不到实际数据，只看到"结构描述"

**用户可验证**：
- 设置页提供"网络活动日志"，列出所有外发请求的目标地址和内容摘要
- 用户可随时切换到本地模式，立即断开所有外部连接

## 9. 异常流程与容错设计

### 9.1 导入阶段

| 异常 | 触发条件 | 用户提示文案 | 恢复操作 |
|------|----------|-------------|----------|
| 格式不支持 | 文件后缀非 .csv | "目前支持 CSV 文件，请转换格式后重新导入" | 保持导入界面，高亮支持格式说明 |
| 编码异常 | 检测到非 UTF-8 编码导致乱码 | 弹出预览窗口："检测到文件可能使用 GBK 编码，预览是否正常？"+ 【看起来正常】【还是乱码，换个编码试试】 | 按 GBK→UTF-8→Latin1 顺序自动尝试，用户确认 |
| 文件过大 | 文件 > 200MB 或行数 > 100 万行 | "文件较大（{size}），建议筛选后导入关键数据" | 提供"仅导入前 N 行"选项 |
| 空文件 | 0 行数据或仅有表头 | "这个文件没有数据内容，请检查是否导出正确" | 返回导入界面 |
| 字段无法识别 | 无日期列或无数值列 | 展示列名列表 + "请确认数据格式，或告诉我这些列代表什么" | 允许用户手动标注列类型 |

### 9.2 提问阶段

| 异常 | 触发条件 | 用户提示文案 | 恢复操作 |
|------|----------|-------------|----------|
| AI 理解错误 | 用户认为结果与预期不符 | 结果区显示"结果不对？"按钮 → 展开选择：【问题理解错了】【数据范围不对】【计算方式不对】 | 根据选择追问细化，重新生成查询 |
| SQL 执行失败 | 生成的 SQL 语法错误或引用不存在的列 | "分析遇到了点问题，正在换个方式重试..." | 自动重试一次。二次失败："抱歉，可以换个说法试试，比如：{推荐相似问题}" |
| 查询超时 | 执行时间 > 30 秒 | "数据量较大，分析还在进行中..."+ 进度条。超 60 秒提示缩小范围 | 【继续等待】【取消并缩小范围】 |
| 查询结果为空 | 返回 0 行 | "没有找到符合条件的数据。可能原因：时间范围无数据 / 筛选条件太严格" | 列出数据的实际时间范围和字段取值范围 |
| 问题超出能力 | 用户问预测/因果类问题 | "目前我擅长数据统计和趋势分析。预测类问题暂时还做不好，怕误导你" | 推荐 3 个相近但能回答的问题 |

### 9.3 结果阶段

| 异常 | 触发条件 | 用户提示文案 | 恢复操作 |
|------|----------|-------------|----------|
| 数据点过多 | 图表数据点 > 10000 | "数据点较多，已自动按{维度}聚合展示" | 自动降采样，提供【查看明细表格】【导出完整数据】 |
| 图表类型不匹配 | AI 选的图表不合适 | 图表右上角常驻"切换图表"按钮 | 用户一键切换，无需重新提问 |
| 数值精度 | 金额出现过多小数位 | 不提示，自动处理 | 金额保留 2 位，百分比保留 1 位，整数不带小数点 |

### 9.4 全局异常

| 异常 | 触发条件 | 用户提示文案 | 恢复操作 |
|------|----------|-------------|----------|
| LLM 服务不可用 | 本地模型未启动或云端 API 不可达 | "AI 引擎未就绪"+ 【检查本地模型状态】或【切换到云端模式】 | 引导到设置页 |
| 应用崩溃恢复 | 上次非正常退出 | "上次分析未正常结束，是否恢复之前的工作？" | 【恢复】加载上次数据和查询历史 / 【重新开始】 |
